# 06 验收标准与风险

- 项目名称：**Kgent V3**
- 文档版本：v3.0
- 更新日期：2026-03-04

## 1. 验收标准（UAT）

### 功能验收

| # | 验收项 | 通过条件 |
|---|---|---|
| UAT-01 | snapshot 可正常打快照 | 执行后 `index.json` 新增一条记录，快照文件存在 |
| UAT-02 | 文件变更被捕获 | 在监控目录新增/修改/删除文件后，两个快照的对比日报中出现对应记录 |
| UAT-03 | report 生成日报 | 输出目录生成 Markdown 文件，包含概览表格和文件明细 |
| UAT-04 | 行级统计正确 | 新增 N 行的文件，日报中 `+N` 显示正确 |
| UAT-05 | 忽略规则生效 | 命中 ignore 规则的文件路径不出现在日报中 |
| UAT-06 | list 展示正确 | 输出包含 ID / 时间 / 标签 / 触发方式 / 文件数 |
| UAT-07 | 无变更时日报正常 | 同一快照对比自身，日报显示变更文件数为 0 |

### 非功能验收

| # | 验收项 | 通过条件 |
|---|---|---|
| UAT-08 | 稳定性 | 连续运行 8 小时，无崩溃，内存无持续增长 |
| UAT-09 | 性能 | 1 万条快照文件记录下，report 生成时长 < 10 秒 |
| UAT-10 | 异常容错 | 单文件读取失败不影响整体快照完成 |
| UAT-11 | 启动易用性 | 从安装到首次 report 输出，全程 < 5 分钟 |

## 2. 验证方案

### 2.1 功能验证

1. 创建测试目录，打初始快照（A）。
2. 新增 / 修改 / 删除若干文件，打第二个快照（B）。
3. 执行 `report --from A --to B`，校验日报内容与实际操作一致。
4. 修改文件至 ignore 目录，重复上述步骤，确认日报中不出现该文件。

### 2.2 稳定性验证

1. 启动 watch，持续运行 8 小时（或使用计划任务模拟登录）。
2. 期间每小时检查内存占用（`Get-Process python | Select WorkingSet64`）。
3. 人为模拟文件读取权限异常，确认 watch 不崩溃。

### 2.3 性能验证

1. 构造含 1 万个文件的测试目录，打快照，计时 snapshot 完成时长。
2. 比较两个大快照生成日报，计时 report 完成时长。

## 3. 风险与约束

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 行级统计非语义化 | 重构场景可能显示大量增删但实际改动少 | 结合文件 hash 变化判断，后续可选 diff 增强 |
| 大文件只存哈希 | 大文件无行级统计 | 调整 `max_file_size_kb` 或在日报中标注 |
| 首次快照耗时 | 超大目录（万级文件）初次扫描慢 | 拆分 `watch_paths` 按项目粒度监控 |
| 快照文件磁盘占用 | 长期运行快照数量增长 | 定期归档或删除旧快照（后续可自动化） |

## 4. 后续迭代建议

| 优先级 | 功能 | 说明 |
|---|---|---|
| 高 | 快照自动清理 | 保留最近 N 天，归档或删除更早快照 |
| 高 | 日报模板自定义 | 支持固定字段（今日完成/问题风险/明日计划） |
| 中 | 按项目分组统计 | 多 watch_path 时分目录展示变更 |
| 中 | 周报 / 月报 | 跨多天快照汇总生成 |
| 低 | Windows 托盘模式 | 系统托盘图标，快速触发快照和报告 |
| 低 | 敏感路径脱敏 | 共享日报时自动隐藏绝对路径前缀 |
