# 02 功能与非功能需求

- 项目名称：**Kgent V3**
- 文档版本：v3.0
- 更新日期：2026-03-04

## 1. 功能需求

### FR-01 配置管理

1. 从 JSON 文件加载所有运行参数。
2. 必要配置项：`watch_paths`、`ignore_dirs`、`ignore_suffixes`、`ignore_patterns`、`snapshot_root`、`output_root`、`auto_snapshot_times`、`max_file_size_kb`。
3. 配置缺失或格式错误时，给出明确错误信息并退出。

### FR-02 快照采集

1. 扫描所有 `watch_paths` 下的文本文件，遵循 ignore 规则过滤。
2. 对每个文件计算 MD5 哈希与行列表并写入快照 JSON 文件。
3. 超过 `max_file_size_kb` 的文件只存哈希，不存行内容。
4. 每次快照生成后更新快照索引（`index.json`）。
5. 支持三种触发方式：
   - 手动：执行 `snapshot` 命令
   - 定时：`watch` 模式按 `auto_snapshot_times` 自动触发
   - 日结：执行 `report` 命令时自动打终点快照

### FR-03 快照对比

1. 支持选取任意两个快照（起点、终点）进行对比。
2. 对比输出每个变更文件的状态（created / modified / deleted）。
3. 对文本文件输出行级统计：added_lines、removed_lines、net_lines。
4. hash 相同的文件跳过，不计入变更。

### FR-04 日报生成

1. 基于快照对比结果生成 Markdown 日报。
2. 日报必须包含：
   - 快照区间（起点时间 ~ 终点时间）
   - 概览表格（变更文件总数 / 各状态数量 / 行变化）
   - 高频变更文件类型 TopN
   - 高频变更目录 TopN
   - 新增 / 修改 / 删除文件明细列表（含行变化）
   - 工作总结草稿（含今日完成 / 问题风险 / 明日计划占位符）
3. `--from` 和 `--to` 省略时，默认取最近两个快照。

### FR-05 命令行接口

| 命令 | 参数 | 说明 |
|---|---|---|
| `snapshot` | `--label`、`--config` | 立即打一个快照 |
| `list` | `--config` | 列出所有快照（ID / 时间 / 标签 / 触发方式 / 文件数）|
| `report` | `--from`、`--to`、`--config` | 选两个快照生成日报 |
| `watch` | `--config` | 持续运行，按配置时间自动打快照 |

### FR-06 Windows 持续运行

1. 支持在 Windows PowerShell / 终端中长期运行。
2. 支持通过 Windows 计划任务实现开机 / 登录自启。
3. 路径处理兼容 Windows 路径格式（使用 `pathlib.Path`）。

## 2. 非功能需求

### NFR-01 轻量化

- 不依赖本地数据库或重型服务，存储仅用 JSON 文件。
- 快照时按需读取文件，不做周期性全量重扫。

### NFR-02 性能

- 监控模式 CPU 均值 < 5%（常规办公开发机）。
- 1 万条日志规模下日报生成时长 < 10 秒。

### NFR-03 稳定性

- 单文件读取异常不中断主流程，捕获并跳过。
- 连续运行 8 小时以上内存无持续增长。

### NFR-04 可维护性

- 模块职责清晰：Config / Snapshot / Diff / Reporter / Scheduler。
- CLI 接口保持稳定，新功能优先通过配置扩展。

### NFR-05 可用性

- 首次安装到首次运行 < 5 分钟。
- 文档提供 Windows 计划任务配置步骤。
