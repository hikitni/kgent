# 10 - 准出指标与质量基线

> **版本**：v1.0 &nbsp;|&nbsp; **日期**：2026-03-05 &nbsp;|&nbsp; **状态**：生效中
>
> 本文档定义 Kgent V3 每次发布前必须达成的量化准出指标（Exit Criteria），
> 作为版本发布、回归验证和持续集成的通过门禁。

---

## 1. 总览

准出指标分为 **5 大类**，共 **28 项**：

| 类别 | 编号前缀 | 指标数 | 说明 |
|---|---|---|---|
| 性能（Performance） | P | 7 | 关键操作的响应速度上限 |
| 稳定性（Stability） | S | 7 | 长时运行、资源占用、容错 |
| 质量（Quality） | Q | 6 | 测试通过率、覆盖率、准确性 |
| 数据完整性（Data Integrity） | D | 3 | 快照与报告的结构正确性 |
| 易用性（Usability） | U | 3 | 安装到产出、可访问性、错误提示 |

> 所有准出指标 **必须全部达标** 方可发布；如有豁免需在发布记录中标注原因与影响评估。

---

## 2. 性能指标（Performance）

| 编号 | 指标 | 准出标准 | 测量方式 | 依据 |
|---|---|---|---|---|
| P-01 | 快照生成速度（1K 文件） | **≤ 5 秒** | 计时 `take_snapshot()`，1,000 个文本文件目录 | — |
| P-02 | 快照生成速度（10K 文件） | **≤ 30 秒** | 计时 `take_snapshot()`，10,000 个文本文件目录 | — |
| P-03 | Diff 计算速度（10K 文件，500 变更） | **≤ 3 秒** | 计时 `diff_snapshots()` | — |
| P-04 | 日报生成速度（10K 文件快照） | **≤ 10 秒** | 端到端 `generate_report()` | NFR-02 |
| P-05 | AI 摘要生成速度（远程 API） | **≤ 30 秒** | 计时 `summarize_report()` / `generate_full_report()` | — |
| P-06 | AI 摘要生成速度（Ollama 本地） | **≤ 60 秒** | 计时 `generate_full_report()` | — |
| P-07 | Web API 响应（非 AI 端点） | **P95 ≤ 500 ms** | 压测 `/api/snapshots`、`/api/reports`、`/api/compare` 等端点 | — |

### 测试环境基线

| 项目 | 规格 |
|---|---|
| CPU | ≥ 4 核（Intel i5 / AMD Ryzen 5 同级） |
| 内存 | ≥ 8 GB |
| 磁盘 | SSD |
| OS | Windows 10/11 64-bit |
| Python | 3.12+ |

## 3. 稳定性指标（Stability）

| 编号 | 指标 | 准出标准 | 测量方式 | 依据 |
|---|---|---|---|---|
| S-01 | 持续运行无崩溃 | **≥ 8 小时** | Watch 模式连续运行，进程不退出 | NFR-03、UAT-08 |
| S-02 | 内存无泄漏 | **8h 内 WorkingSet 增量 < 20 MB** | 每小时采样 `Get-Process` 的 `WorkingSet64` | NFR-03 |
| S-03 | Watch 空闲 CPU | **均值 < 5%** | 1 小时 `Get-Process` 周期采样 | NFR-02 |
| S-04 | Watch 空闲内存 | **< 150 MB** | 运行时峰值 `WorkingSet64` 监控 | 01-概述与目标 |
| S-05 | 单文件故障容错 | **不可读文件不导致崩溃** | 注入锁定 / 二进制 / 权限不足文件，快照仍正常完成 | NFR-03、UAT-10 |
| S-06 | SSE 连接稳定性 | **1h 内 0 断连** | 保持 SSE 客户端连接，验证 25s 心跳持续到达 | — |
| S-07 | 优雅关闭 | **Watch 停止 ≤ 5 秒** | 计时从 `watch_stop()` 调用到工作线程退出 | — |

## 4. 质量指标（Quality）

| 编号 | 指标 | 准出标准 | 测量方式 | 当前值 |
|---|---|---|---|---|
| Q-01 | 单元测试通过率 | **100%** | `python -m pytest tests/ --tb=short`，退出码 0 | 276/276 ✅ |
| Q-02 | 代码行覆盖率 | **≥ 85%** | `python -m pytest tests/ --cov=daily_reporter` | 86% ✅ |
| Q-03 | 功能验收项（UAT）全通过 | **UAT-01 ~ UAT-07 共 7 项** | 逐项执行功能测试 | 7/7 ✅ |
| Q-04 | Diff 零误报 | **同一快照对比 → 0 差异** | `diff_snapshots(snap, snap)` 返回空列表 | ✅ |
| Q-05 | 行数统计零偏差 | **增加 N 行 → `added_lines == N`** | 已知变更文件手工验证 | ✅ |
| Q-06 | Ignore 规则无漏出 | **被忽略文件不出现在快照与报告中** | 创建匹配所有 ignore 模式的文件，验证缺失 | ✅ |

## 5. 数据完整性指标（Data Integrity）

| 编号 | 指标 | 准出标准 | 测量方式 |
|---|---|---|---|
| D-01 | 快照文件一致性 | **meta（`.json`）与 content（`.content.json`）一一对应** | 遍历所有快照，验证 `meta.paths` ⊆ `content.keys()` |
| D-02 | AI 报告结构完整性 | **包含 5 个必要章节**：概述 / 详情 / 代码变更 / 问题 / 计划 | 解析 AI 输出 Markdown 标题 |
| D-03 | 报告文件可读性 | **非空且为合法 Markdown** | 生成报告 > 0 字节，可被 Markdown 解析器正常渲染 |

## 8. CI/CD 集成建议

准出指标应嵌入到持续集成流水线中，推荐执行顺序：

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  Q-01 ~ Q-06 │────▶│  P-01 ~ P-04 │────▶│  D-01 ~ D-03 │
│  单元测试     │     │  性能基准     │     │  数据校验     │
│  覆盖率门禁   │     │  (benchmark)  │     │  (验证脚本)   │
└──────────────┘     └──────────────┘     └──────────────┘
       │                                         │
       ▼                                         ▼
┌──────────────┐                          ┌──────────────┐
│  S-05        │                          │  U-02        │
│  容错测试     │                          │  端点健康检查  │
└──────────────┘                          └──────────────┘
```

### 门禁规则

- **阻断级（Block）**：Q-01、Q-02、Q-04、Q-05、Q-06 — 不通过则禁止合并 / 发布
- **警告级（Warn）**：P-01 ~ P-07、S-05 — 超标时产生警告，需人工评审
- **手工级（Manual）**：S-01 ~ S-04、S-06、S-07、U-01、U-03、D-02 — 发布前人工执行
