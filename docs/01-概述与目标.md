# 01 概述与目标

- 项目名称：**Kgent V3**
- 文档版本：v3.0
- 更新日期：2026-03-04

## 1. 背景

开发者日常会对大量本地文件进行频繁修改，手工整理日报耗时且容易遗漏改动细节。现有工具要么依赖 Git（仅覆盖已提交内容），要么依赖云端服务（引入隐私风险与复杂度）。

**Kgent V3** 的目标是提供一个**轻量、稳定、可长期运行在 Windows** 的纯本地工具，通过**快照对比**采集文件变更，在工作日结束时快速生成日报草稿，并可选接入大模型（ZhipuAI / OpenAI / Ollama）生成口语化 AI 日报，从而大幅降低日常总结成本。

## 2. 核心目标

| 目标 | 量化指标 |
|---|---|
| 轻量化 | 监控进程 CPU 均值 < 5%，内存 < 150 MB |
| 持续运行 | Windows 下稳定运行 8 小时以上，单文件异常不中断 |
| 快速总结 | 执行 report 命令后 1 分钟内输出可提交初稿 |

## 3. 范围

### 范围内

- 对指定目录进行定时或手动快照
- 对比任意两个快照，输出文件级变更摘要（新增 / 修改 / 删除）
- 行级增删统计（added_lines / removed_lines / net_lines）
- 生成 Markdown 格式日报，包含概览、文件明细、工作总结草稿
- 通过 JSON 配置控制监控范围、忽略规则、自动快照时间

### 范围外

- 不做语义级代码理解（函数职责、业务意图）
- 不分析二进制文件内容
- 不提供云端存储、多人协作、Web 后台
- 不替代 Git 提交历史

## 4. 目标用户与典型流程

**目标用户**：需要产出工作日报的个人开发者、测试或运维人员

**推荐每日流程**

```
开机/上班
  └─ watch 自动在 06:00 打一个快照（label: morning）

工作中（可选）
  └─ 随时手动打快照，记录里程碑状态

下班前
  └─ 执行 report → 自动打日结快照 + 生成日报
       └─ 在日报草稿中补充业务描述后提交
```

若当天电脑未在 06:00 开机，开机后手动补打：

```bash
python main.py snapshot --label morning
```
