# 04 数据规范

- 项目名称：**Kgent V3**
- 文档版本：v3.0
- 更新日期：2026-03-04

## 1. 目录结构

```
.daily_reporter/          ← snapshot_root（可配置）
  index.json              ← 快照索引
  snapshots/
    snapshot-YYYYMMDD-HHmmss.json   ← 每个快照独立文件

reports/                  ← output_root（可配置）
  report-{A.id}-to-{B.id}.md       ← 日报文件
```

## 2. 快照文件规范

- 格式：JSON
- 命名：`snapshot-YYYYMMDD-HHmmss.json`
- 存放：`snapshot_root/snapshots/`

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | string | ✓ | 唯一标识，格式 `YYYYMMDD-HHmmss` |
| `timestamp` | string | ✓ | ISO 8601 时间，精确到秒 |
| `label` | string | ✓ | 语义标签：morning / noon / afternoon / evening / report / manual |
| `trigger` | string | ✓ | 触发方式：auto / manual / report |
| `file_count` | int | ✓ | 本次扫描文件总数 |
| `files` | object | ✓ | 文件路径 → 内容映射 |
| `files[path].hash` | string | ✓ | 文件内容 MD5 |
| `files[path].lines` | array\|null | ✓ | 文件行列表；大文件时为 null |

## 3. 快照索引规范

- 格式：JSON 数组
- 存放：`snapshot_root/index.json`
- 说明：仅存元数据，不含文件内容，用于 `list` 命令快速展示

```json
[
  {
    "id": "20260304-060000",
    "timestamp": "2026-03-04T06:00:00",
    "label": "morning",
    "trigger": "auto",
    "file_count": 42
  }
]
```

## 4. 日报文件规范

- 格式：Markdown
- 命名：`report-{起点ID}-to-{终点ID}.md`
- 存放：`output_root/`

### 日报章节结构

| 章节 | 内容 |
|---|---|
| 元信息 | 快照起点、终点、生成时间 |
| 一、今日概览 | 变更文件数 / 各状态数量 / 增删行数表格 |
| 二、高频变更文件类型 | 后缀名 Top5 及文件数 |
| 三、高频变更目录 | 目录 Top5 及文件数 |
| 四、新增文件 | 列表（最多 20 条，含行变化） |
| 五、修改文件 | 列表（最多 20 条，含行变化） |
| 六、删除文件 | 列表 |
| 七、工作总结 | 自动草稿 + 今日完成 / 问题风险 / 明日计划占位符 |

## 5. 配置文件规范

- 格式：JSON
- 默认路径：`config.json`（可通过 `--config` 指定）

```json
{
  "watch_paths": ["./"],
  "ignore_dirs": [".git", "node_modules", "__pycache__", ".daily_reporter", "reports"],
  "ignore_suffixes": [".log", ".tmp", ".swp", ".pyc"],
  "ignore_patterns": ["\\.idea\\\\", "\\.vscode\\\\"],
  "snapshot_root": ".daily_reporter",
  "output_root": "reports",
  "auto_snapshot_times": ["06:00", "18:00"],
  "max_file_size_kb": 1024
}
```
