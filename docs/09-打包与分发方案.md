# 09 - 打包与分发方案

> 本文档描述 KgentV3 如何打包为可分发的 Windows 可执行程序压缩包，使接收者无需安装 Python 即可使用。

---

## 1. 方案概述

采用 **PyInstaller** 将整个 Python 项目（含 FastAPI Web 资源）打包为单个 `.exe` 文件，并连同配置模板、AI Prompt 模板、启动脚本和使用说明一起压缩为 ZIP 压缩包。

### 1.1 发行包结构

```
KgentV3-release.zip
  └── KgentV3/
      ├── KgentV3.exe              # 主程序（单文件 ~40-80MB）
      ├── config.example.json      # 配置模板
      ├── ai_report_prompt.md      # AI 提示词模板
      ├── start-web.bat            # 双击启动 Web 界面
      ├── start-cli.bat            # 双击启动命令行交互菜单
      └── 启动说明.md              # 面向使用者的操作手册
```

### 1.2 使用者视角

1. 解压 ZIP 到任意目录
2. 复制 `config.example.json` → `config.json`，编辑 `watch_paths` 等参数
3. 双击 `start-web.bat` 或 `start-cli.bat` 即可使用

---

## 2. 打包工具链

| 工具 | 版本要求 | 用途 |
|---|---|---|
| Python | ≥ 3.10 | 运行环境（仅打包机需要） |
| PyInstaller | ≥ 6.0 | 将 Python 项目冻结为 .exe |
| PowerShell | ≥ 5.1 | 执行 `build.ps1` 打包脚本 |

---

## 3. 打包脚本 `build.ps1` 工作流

```
┌─────────────────────────────────────────────────┐
│               build.ps1  执行流程               │
├─────────────────────────────────────────────────┤
│                                                 │
│  1. 检查 / 自动安装 PyInstaller                 │
│              ↓                                  │
│  2. 清理旧的 dist/ build/ *.spec 产物           │
│              ↓                                  │
│  3. PyInstaller --onefile 打包                  │
│     ├── --add-data  web/static/                 │
│     ├── --add-data  web/templates/              │
│     ├── --add-data  ai_report_prompt.md         │
│     └── --hidden-import  uvicorn / 子模块       │
│              ↓                                  │
│  4. 组装发行目录  dist/KgentV3/                 │
│     ├── KgentV3.exe                             │
│     ├── config.example.json                     │
│     ├── ai_report_prompt.md                     │
│     ├── 启动说明.md                             │
│     ├── start-web.bat   (脚本内动态生成)        │
│     └── start-cli.bat   (脚本内动态生成)        │
│              ↓                                  │
│  5. Compress-Archive → KgentV3-release.zip      │
│              ↓                                  │
│  6. 输出打包结果摘要                            │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 3.1 关键 PyInstaller 参数说明

| 参数 | 说明 |
|---|---|
| `--onefile` | 将所有依赖打包为单个 exe，解压到临时目录后运行 |
| `--console` | 保留命令行窗口，用于显示日志和交互菜单 |
| `--add-data src;dst` | 将静态资源/模板嵌入 exe 内部（运行时解压到 `sys._MEIPASS`） |
| `--hidden-import` | 显式声明动态导入的模块（如 uvicorn 内部按协议自动加载的子模块） |

### 3.2 资源路径兼容处理

PyInstaller 打包后，嵌入资源的根路径从源码目录变为 `sys._MEIPASS` 临时目录。项目中通过 `pathlib.Path(__file__).parent` 定位资源的代码需要兼容此路径。FastAPI 的 `StaticFiles` 和 `Jinja2Templates` 挂载点应使用相对于模块文件位置的路径，确保开发态与打包态均可正确加载。

## 4. hidden-import 清单

以下模块需要在 PyInstaller 打包时显式声明，否则运行时会出现 `ModuleNotFoundError`：

```
daily_reporter.*              # 项目全部子模块
daily_reporter.web.*          # Web 子模块
daily_reporter.web.routes.*   # 路由子模块
uvicorn                       # ASGI 服务器
uvicorn.logging               # 日志模块
uvicorn.protocols.http.auto   # HTTP 协议自动选择
uvicorn.protocols.websockets.auto
uvicorn.lifespan.on           # ASGI 生命周期处理
```

---

## 5. 打包执行命令

在项目根目录打开 PowerShell，执行：

```powershell
.\build.ps1
```

产出物位于：

```
dist/
├── KgentV3.exe              # 独立可执行文件
├── KgentV3/                 # 发行目录（含全部附件）
└── KgentV3-release.zip      # 最终分发压缩包
```

## 6. 注意事项与已知限制

### 6.1 首次启动较慢

`--onefile` 模式下，exe 启动时需要先将内嵌资源解压到临时目录（`%TEMP%\_MEIxxxxx`），首次启动可能需要 3-8 秒。后续启动会因系统缓存而加快。

### 6.2 杀毒软件误报

部分杀毒软件可能对未签名的 PyInstaller exe 产生误报。解决方案：
- 在杀毒软件中添加白名单
- 如需正式分发，可购买代码签名证书对 exe 进行签名

### 6.3 配置文件路径

打包后的 exe 默认在 **当前工作目录** 查找 `config.json`，因此 `start-web.bat` 中未指定绝对路径，exe 与配置文件需要在同一目录下。

### 6.4 体积优化

如对体积敏感，可考虑：
- 使用 `--exclude-module` 排除测试依赖（pytest、httpx 等）
- 使用 UPX 压缩：`--upx-dir <upx路径>`
- 切换至 `--onedir` 模式（目录分发，启动更快但文件更多）

## 7. 备选分发方案

| 方案 | 适用场景 | 详见 |
|---|---|---|
| ZIP + Python 环境 | 对方已有 Python | 直接发送源码 + `requirements.txt` |
| Docker 容器 | 服务器 / 团队共用 | 编写 `Dockerfile`，文件挂载监控目录 |
| PyPI 发布 | 开源分发 | 添加 `pyproject.toml`，`pip install kgent-v3` |
